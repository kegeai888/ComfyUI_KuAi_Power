# 批量处理实时监控功能使用指南

## 概述

新增的**实时监控节点**可以让你在批量处理运行时，随时查看最新的任务状态，无需等待全部完成。

## 核心特性

### 1. 实时状态更新
- 并发处理器在处理过程中，实时更新任务状态到文件
- 状态包括：pending（等待）、processing（处理中）、completed（完成）、failed（失败）

### 2. 轮询监控
- 添加 `📡 批量处理实时监控` 节点
- 重复执行节点即可刷新日志
- 最新的任务显示在前面（倒序）

### 3. 详细信息
- 总进度（已完成/总数）
- 成功率统计
- 每个任务的详细状态
- 错误信息
- 保存路径

## 使用方法

### 方法1：在工作流中添加监控节点

1. **添加监控节点**
   - 在 ComfyUI 中搜索 `📡 批量处理实时监控`
   - 添加到工作流画布

2. **配置参数**
   - `最多显示任务数`：默认 20，可调整为 5-100

3. **连接到 ShowAny 节点**
   ```
   📡 批量处理实时监控
     └─ 实时日志 (STRING) → ShowAny 节点
   ```

4. **运行批量处理**
   - 启动批量工作流
   - 批量处理器开始运行

5. **查看实时日志**
   - 重复执行监控节点（右键 → Queue Prompt）
   - 每次执行都会读取最新状态
   - ShowAny 节点会显示最新日志

### 方法2：独立监控工作流

创建一个独立的监控工作流：

```
📡 批量处理实时监控
  └─ 实时日志 → ShowAny
```

在批量处理运行时，重复执行这个工作流即可查看最新状态。

## 使用技巧

### 1. 快速刷新

在 ComfyUI 中：
1. 右键点击监控节点
2. 选择 "Queue Prompt"
3. 或使用快捷键（通常是 Ctrl+Enter）

### 2. 自动刷新（手动方式）

虽然不是真正的自动刷新，但你可以：
1. 每隔 10-30 秒手动执行一次监控节点
2. 观察任务进度变化
3. 发现问题及时处理

### 3. 多窗口监控

1. 在一个浏览器窗口运行批量处理
2. 在另一个窗口打开监控工作流
3. 定期刷新监控窗口查看状态

### 4. 调整显示数量

- 如果任务很多，可以减少 `最多显示任务数` 到 10
- 如果任务较少，可以增加到 50 或更多
- 只显示最新的任务，避免信息过载

## 故障排查

### 问题1：监控节点显示"暂无批量处理任务运行"

**原因**：
- 批量处理还未开始
- 状态文件不存在或已清除

**解决**：
1. 确保批量处理工作流正在运行
2. 检查状态文件是否存在：`ls -la /root/ComfyUI/temp/batch_process_state.json`

### 问题2：状态不更新

**原因**：
- 并发处理器版本过旧
- 状态管理器未正确初始化

**解决**：
1. 确保使用最新版本的并发处理器
2. 重启 ComfyUI
3. 检查控制台是否有错误信息

### 问题3：日志显示不完整

**原因**：
- `最多显示任务数` 设置过小

**解决**：
1. 增加 `最多显示任务数` 参数
2. 或者只关注最新的任务
