# 批量处理详细日志功能使用指南

## 概述

新增的批量处理详细日志功能可以让你在批量运行任务时，实时看到每个任务的详细运行状态，包括：
- 任务进度（已完成/总数）
- 每个任务的状态（成功/失败）
- 详细的错误信息
- 视频保存路径
- 成功率统计

## 功能特性

### 1. 实时控制台日志

并发处理器会在控制台输出详细的实时日志：

```

[GrokCSVConcurrent] 保存目录: output/grok

[GrokCSVConcurrent] [1] 已提交 task_id=abc123
[GrokCSVConcurrent] [2] 已提交 task_id=def456
...
[GrokCSVConcurrent] [1] 进行中 10/1200s
[GrokCSVConcurrent] [2] 进行中 10/1200s
...
[GrokCSVConcurrent] [1] 完成，下载中...
[GrokCSVConcurrent] [2] 完成，下载中...
```

### 2. 格式化详细报告

新增的 `📊 批量处理详细日志` 节点会生成美化的报告：

## 使用方法

### 方法1：使用更新后的批量工作流（推荐）

批量工作流已自动添加了日志显示节点，直接使用即可：

1. 打开批量工作流：
   - `/root/ComfyUI/user/default/workflows/0-批量veo3-grok3/grok批量图生视频.json`
   - `/root/ComfyUI/user/default/workflows/0-批量veo3-grok3/grok批量文生视频.json`
   - `/root/ComfyUI/user/default/workflows/0-批量veo3-grok3/veo3批量图生视频.json`
   - `/root/ComfyUI/user/default/workflows/0-批量veo3-grok3/veo3批量文生视频.json`

2. 运行工作流

3. 查看日志：
   - **控制台**：实时查看任务进度
   - **日志节点**：查看格式化的详细报告

### 方法2：手动添加日志节点

如果你有自己的批量工作流，可以手动添加日志节点：

1. 在 ComfyUI 中打开你的批量工作流

2. 添加 `📊 批量处理详细日志` 节点（在 `KuAi/Utils` 分类下）

3. 连接节点：
   - 将并发处理器的 `详细报告JSON` 输出连接到日志节点的 `处理报告JSON` 输入

4. 运行工作流

## 节点说明

### 📦 Grok/Veo3 CSV 并发批量处理器（已更新）

**新增输出**：
- `详细报告JSON`：包含所有任务详细信息的 JSON 数据

**输出格式**：
```json
{
  "total": 10,
  "success": 8,
  "failed": 2,
  "tasks": [
    {
      "idx": 1,
      "row": 2,
      "status": "completed",
      "prompt": "电商直播...",
      "video_url": "https://...",
      "local_path": "output/grok/grok_1_a1b2c3d4.mp4",
      "error": ""
    },
    ...
  ]
}
```

### 📊 批量处理详细日志

**输入**：
- `处理报告JSON`：来自并发处理器的详细报告
- `详细模式`：是否显示详细信息（默认：True）

**输出**：
- `格式化日志`：美化后的文本报告

**功能**：
- 自动格式化 JSON 报告
- 添加状态图标（✓ ✗ ⟳ ○）
- 计算成功率
- 分类显示成功/失败任务
- 显示详细错误信息

## 日志级别

### 控制台日志（实时）

- 任务提交：`[GrokCSVConcurrent] [1] 已提交 task_id=abc123`
- 轮询进度：`[GrokCSVConcurrent] [1] 进行中 10/1200s`
- 任务完成：`[GrokCSVConcurrent] [1] 完成，下载中...`
- 任务失败：`[GrokCSVConcurrent] [1] ✗ 超时 (1200s)`

### 格式化报告（完成后）

- 总体统计：总计、成功、失败、成功率
- 成功任务列表：任务ID、提示词、保存路径
- 失败任务列表：任务ID、提示词、错误信息

## 故障排查

### 问题1：看不到日志节点

**原因**：节点未注册

**解决**：
1. 重启 ComfyUI
2. 检查控制台是否有 `[ComfyUI_KuAi_Power]` 相关错误
3. 运行诊断：`python diagnose.py`

### 问题2：日志节点报错

**原因**：JSON 格式错误

**解决**：
1. 确保并发处理器已更新（有第三个输出）
2. 检查连接是否正确（连接到 `详细报告JSON` 输出）
3. 查看控制台错误信息

### 问题3：控制台没有实时日志

**原因**：ComfyUI 控制台未打开

**解决**：
1. 如果使用命令行启动，日志会直接显示在终端
2. 如果使用桌面版，打开 ComfyUI 的控制台窗口
3. 检查 ComfyUI 的日志设置

## 最佳实践

1. **开发调试**：使用详细模式（verbose=True）查看所有信息
2. **生产运行**：可以关闭详细模式（verbose=False）只看统计
3. **大批量任务**：建议调整批次大小（batch_size）避免过载
4. **错误处理**：失败任务会在报告中详细列出，可以单独重试
