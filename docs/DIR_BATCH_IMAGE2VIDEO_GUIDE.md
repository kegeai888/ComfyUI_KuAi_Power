# 目录批量图生视频使用指南

## 概述

**目录批量图生视频节点**是一个完整闭环的一键式解决方案，用于将本地目录中的多张图片批量转换为视频。

### 核心特性

- ✅ **完整闭环**：单节点完成全流程（扫描 → 上传 → 生成 → 下载）
- ✅ **并发上传**：多线程并发上传图片，速度快
- ✅ **并发生成**：批量并发提交视频生成任务
- ✅ **自动轮询**：自动轮询任务状态直到完成
- ✅ **自动下载**：生成完成后自动下载 MP4 到本地
- ✅ **智能排序**：按文件名自然数字排序（1.png, 2.png, ..., 10.png）

### 支持平台

- **Grok** - `GrokDirBatchImage2Video`
- **Veo3** - `VeoDirBatchImage2Video`

---

## 使用步骤

### 1. 准备图片目录

将图片放在指定目录下，例如：

```
/root/ComfyUI/input/grok/demo1/
├── 1.png
├── 2.png
├── 3.jpg
├── 4.png
└── ...
```

**要求**：
- 图片文件名建议使用数字命名（1, 2, 3, ...）
- 支持格式：`.png`, `.jpg`, `.jpeg`, `.webp`, `.bmp`, `.gif`
- 图片会按文件名自然排序（1 < 2 < 10 < 20）

### 2. 在 ComfyUI 中添加节点

**Grok**：搜索 `⚡ Grok 目录批量图生视频（一键闭环）`
**Veo3**：搜索 `⚡ Veo3 目录批量图生视频（一键闭环）`

或使用快捷面板（Ctrl+Shift+K）在 Grok/Veo3 分类下找到。

### 3. 配置参数

#### 必需参数

| 参数 | 说明 | 示例 |
|------|------|------|
| **图片目录路径** | 本地图片目录的完整路径 | `/root/ComfyUI/input/grok/demo1` |
| **提示词** | 视频生成提示词（所有图片共用） | `一个产品展示视频，镜头缓慢推进` |
| **API密钥** | API密钥（留空使用环境变量 `KUAI_API_KEY`） | 留空或填写密钥 |

#### 可选参数（Grok）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **模型** | `grok-video-3 (6秒)` | 选择 Grok 模型（6秒/10秒/15秒） |
| **宽高比** | `3:2` | 视频宽高比（2:3, 3:2, 1:1） |
| **分辨率** | `1080P` | 视频分辨率（720P, 1080P） |
| **优化提示词** | `true` | 自动优化并翻译提示词 |
| **视频保存目录** | `output/grok` | 视频保存目录（相对 ComfyUI 根目录） |
| **输出前缀** | `grok` | 输出文件名前缀 |
| **并发批次大小** | `5` | 每批并发任务数（图生视频阶段） |
| **上传并发数** | `5` | 并发上传线程数（上传阶段） |
| **最大图片数** | `100` | 最大处理图片数 |

#### 可选参数（Veo3）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **模型** | `veo3.1` | 选择 Veo3 模型 |
| **宽高比** | `9:16` | 视频宽高比（16:9, 9:16） |
| **优化提示词** | `true` | 自动优化并翻译提示词 |
| **启用超分** | `true` | 启用超分以提升视频质量 |
| **视频保存目录** | `output/veo3` | 视频保存目录（相对 ComfyUI 根目录） |
| **输出前缀** | `veo3` | 输出文件名前缀 |
| **并发批次大小** | `5` | 每批并发任务数（图生视频阶段） |
| **上传并发数** | `5` | 并发上传线程数（上传阶段） |
| **最大图片数** | `100` | 最大处理图片数 |

### 4. 执行工作流

点击 **Queue Prompt** 执行。

## 执行流程

节点会按以下三个阶段执行：

### 阶段 1：扫描目录

```
[GrokDirBatch] 阶段 1/3：扫描目录
============================================================
[GrokDirBatch] 找到 10 张图片
```

- 扫描指定目录
- 按文件名自然排序
- 限制最大图片数

### 阶段 2：并发上传

```
[GrokDirBatch] 阶段 2/3：并发上传（5 线程）
============================================================
[GrokDirBatch] [1] 上传成功: 1.png
[GrokDirBatch] [2] 上传成功: 2.png
...
[GrokDirBatch] 上传完成: 成功 10/10
```

- 使用线程池并发上传
- 每张图片上传到图床获取 CDN URL
- 失败的图片会跳过，不影响其他图片

### 阶段 3：并发图生视频 + 轮询 + 下载

```
[GrokDirBatch] 阶段 3/3：并发图生视频（每批 5 路）
============================================================
[GrokDirBatch] 批次 1/2：提交 5 个任务
[GrokDirBatch] [1] 已提交 task_id=xxx
[GrokDirBatch] [2] 已提交 task_id=yyy
...
[GrokDirBatch] [1] 进行中 10/1200s
[GrokDirBatch] [2] 进行中 10/1200s
...
[GrokDirBatch] [1] 生成完成，下载中...
[GrokDirBatch] [2] 生成完成，下载中...
```

- 按批次并发提交任务
- 每个任务独立轮询状态
- 完成后自动下载 MP4 到本地

## 常见问题

### Q1: 目录不存在或没有图片？

**错误**：`目录不存在: /path/to/dir` 或 `目录中没有找到图片`

**解决**：
- 检查目录路径是否正确
- 确保目录中有支持的图片格式
- 检查文件权限

### Q2: 上传失败？

**错误**：`上传失败 HTTP 500`

**解决**：
- 检查网络连接
- 尝试更换图床地址
- 检查图片文件是否损坏

### Q3: 生成超时？

**错误**：`超时 (1200s)`

**解决**：
- 增加 `最大等待时间` 参数
- 检查 API 服务状态
- 简化提示词

### Q4: API Key 未配置？

**错误**：`API Key 未配置`

**解决**：
```bash
# 方法 1：环境变量
export KUAI_API_KEY=your_key_here

# 方法 2：节点参数
在节点的 API密钥 参数中填写
```

### Q5: 如何控制并发数？

**调整参数**：
- **上传并发数**：控制上传阶段的并发线程数（建议 5-10）
- **并发批次大小**：控制视频生成阶段的并发任务数（建议 5-10）

**注意**：并发数过高可能导致 API 限流或服务器压力过大。

## 性能优化建议

### 1. 上传阶段

- **并发数**：根据网络带宽调整（5-10 线程）
- **图片格式**：JPEG 压缩率高，上传快
- **图片质量**：90 是质量和大小的平衡点

### 2. 生成阶段

- **批次大小**：5-10 路并发较为稳定
- **轮询间隔**：Grok 10秒，Veo3 15秒
- **超时时间**：根据模型时长调整（6秒视频建议 600s，15秒视频建议 1200s）

### 3. 目录组织

```
/root/ComfyUI/input/
├── grok/
│   ├── product_a/    # 产品 A 的图片
│   ├── product_b/    # 产品 B 的图片
│   └── demo1/        # 演示图片
└── veo3/
    ├── scene_1/      # 场景 1 的图片
    └── scene_2/      # 场景 2 的图片
```

## 与旧工作流的对比

### 旧工作流（3 节点）

```
BatchImageUploader → BatchImageToCSVTask（GrokBatchImageToCSVTask / VeoBatchImageToCSVTask） → CSVConcurrentProcessor（GrokCSVConcurrentProcessor / VeoCSVConcurrentProcessor）
```

**缺点**：
- 需要连接 3 个节点
- 上传是串行的（慢）
- 中间节点只是格式转换（冗余）

### 新工作流（1 节点）

```
GrokDirBatchImage2Video / VeoDirBatchImage2Video
```

**优点**：
- 单节点完成全流程
- 上传并发（快）
- 配置简单，易用

## 技术细节

### 图片上传

- **端点**：`https://imageproxy.zhongzhuan.chat/api/upload`
- **方法**：`POST multipart/form-data`
- **并发**：使用 `ThreadPoolExecutor` 并发上传
- **返回**：CDN URL

### 视频生成

**Grok**：
- **创建**：`GrokCreateVideo.create(image_urls=url)`
- **查询**：`GrokQueryVideo.query(task_id)`
- **轮询**：手动 while 循环 + `time.sleep(poll_interval)`

**Veo3**：
- **创建**：`VeoImage2Video.create(image_1=url)`
- **查询**：`VeoQueryTask.query(task_id, wait=False)`
- **轮询**：手动 while 循环 + `time.sleep(poll_interval)`

### 视频下载

- **方法**：流式下载 `requests.get(stream=True)`
- **文件名**：`{prefix}_{序号}_{URL_MD5[:8]}.mp4`
- **路径**：相对 ComfyUI 根目录
