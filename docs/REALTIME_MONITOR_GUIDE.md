# 实时批量处理监控使用指南

## 概述

实时批量处理监控节点（`RealtimeBatchMonitor`）提供了真正的实时监控体验，通过 WebSocket 自动推送批处理进度，无需手动刷新。

## 功能特性

### 核心功能
- ✅ **真正实时**：每 3 秒自动更新，无需手动操作
- ✅ **详细日志**：显示完整的 API 调用、轮询过程、下载进度、错误详情
- ✅ **全面统计**：总体进度、成功率、平均耗时、预计剩余时间
- ✅ **浮动面板**：固定在右上角，可拖动、可折叠
- ✅ **自动隐藏**：任务完成后 3 秒自动隐藏

### 技术架构
- **后端**：后台线程 + WebSocket 推送
- **前端**：浮动面板 + 实时 UI 更新
- **状态管理**：增强的 `BatchProcessState` 支持详细日志和统计

## 使用方法

### 1. 添加节点到工作流

在 ComfyUI 中：
1. 按 `Ctrl+Shift+K` 打开快速面板
2. 选择 "🛠️ 工具节点" 分类
3. 找到 "📡 实时批量监控" 节点
4. 添加到画布

### 2. 配置参数

| 参数 | 说明 | 默认值 | 范围 |
|------|------|--------|------|
| **启用监控** | 启用/禁用实时监控 | True | Boolean |
| **刷新间隔** | 自动刷新间隔（秒） | 3.0 | 1.0 - 10.0 |
| **最多显示任务数** | 最多显示的任务数 | 10 | 5 - 50 |
| **最多显示日志条数** | 最多显示的日志条数 | 50 | 10 - 200 |

### 3. 连接工作流

**重要**：`RealtimeBatchMonitor` 节点是独立运行的，不需要连接到其他节点。

推荐工作流结构：
```
CSVBatchReader → GrokCSVConcurrentProcessor
                 ↓
            (自动保存视频)

RealtimeBatchMonitor (独立运行，监控状态文件)
```

### 4. 执行工作流

1. 启动批量处理工作流（如 Grok 批量图生视频）
2. 执行 `RealtimeBatchMonitor` 节点
3. 浮动面板自动出现在右上角
4. 每 3 秒自动更新进度

## 统计信息说明

### 平均耗时
- 计算所有已完成任务的平均处理时间
- 从任务开始到完成的总耗时
- 用于预估剩余任务的处理时间

### 成功率
- 成功任务数 / 已完成任务数 × 100%
- 颜色指示：
  - 绿色：≥ 80%
  - 黄色：< 80%

### 预计剩余时间
- 基于平均耗时 × 剩余任务数
- 动态更新，随着任务进行越来越准确

## 与现有节点的区别

| 特性 | BatchProcessMonitor | RealtimeBatchMonitor |
|------|---------------------|----------------------|
| **更新方式** | 手动重新执行 | 自动推送（每 3 秒） |
| **显示方式** | 节点输出 | 浮动面板 |
| **详细日志** | 无 | 完整日志输出 |
| **统计信息** | 基础统计 | 详细统计（平均耗时、预计剩余） |
| **用户体验** | 需要手动操作 | 完全自动化 |

## 故障排除

### 问题 1：面板不出现
**原因**：WebSocket 连接失败或节点未启用

**解决方法**：
1. 检查 ComfyUI 控制台是否有错误信息
2. 确认 `enable` 参数设置为 `True`
3. 刷新浏览器页面

### 问题 2：进度不更新
**原因**：后台线程未启动或状态文件未更新

**解决方法**：
1. 检查 ComfyUI 控制台日志：`[RealtimeBatchMonitor] 监控线程已启动`
2. 确认批量处理器正在运行
3. 检查状态文件：`ComfyUI/temp/batch_process_state.json`

### 问题 3：日志显示不完整
**原因**：日志条数限制

**解决方法**：
1. 增加 `max_logs` 参数（最大 200）
2. 查看完整日志文件（如果有）

### 问题 4：面板位置不对
**原因**：拖动后位置保存

**解决方法**：
1. 刷新浏览器页面重置位置
2. 手动拖动到合适位置

## 性能说明

### 资源占用
- **CPU**：< 1%（后台线程轮询）
- **内存**：< 10MB（状态文件 + 日志缓存）
- **网络**：< 1KB/s（WebSocket 推送）

### 性能优化
- 日志使用循环缓冲区（FIFO），最多保存 1000 条
- 前端只渲染最新 20 条日志
- WebSocket 推送使用节流机制

```
test/
└── test_realtime_monitor.py  # 测试脚本
```

### 扩展开发
如果需要为其他处理器添加实时监控支持：

1. 在处理器中使用 `BatchProcessState`
2. 在关键节点调用 `state_manager.add_log()`
3. 无需修改 `RealtimeBatchMonitor` 节点

示例：
```python
from ..Utils.batch_state import BatchProcessState

state_manager = BatchProcessState()
state_manager.start_session("my_session", total=10)
state_manager.add_log(1, "INFO", "任务开始")
state_manager.update_task(1, "processing", task_id="abc123")
state_manager.add_log(1, "DEBUG", "轮询中...")
state_manager.update_task(1, "completed", video_url="https://...")
state_manager.add_log(1, "INFO", "任务完成")
```
